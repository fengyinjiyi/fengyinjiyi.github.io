微信小程序async/await函数使用报错

发表于 2019-07-28 | 分类于 前端 ， 微信小程序 | 评论数 | 热度: | 字数统计: 1.3k

  最近在做微信小程序的云开发，突然发现小程序的API都是异步的，这样的话如果有时候多个API的请求数据需要有先后关系的，那么只能将后面API在上一个API的回调函数中调用，不仅显得代码不优雅（作者有点追求代码简洁的癖好！），而且代码量也大了。
  解决办法就是用 async/await 来将异步的API请求变成同步。

场景描述

  再描述一下需求场景吧，例如分页功能，微信小程序云开发实现分页的话，如果要在页面上显示总页数，则需要调用 count() 先获取数据的总条数，根据得到的总条数计算出总页数，然后用 limit() 获取相应页数的数据。在这种需求场景下，count() 和 limit() 两个API写在同一个函数体中，因为都是异步的，就算先执行 count() 也不一定比 limit() 先返回，但是我们就是需要 count() 返回之后再执行 limit() ，这个时候需要用到 async/await 来实现同步，这样就一定能保证 count() 返回之后再去执行 limit() 来获取数据。

  这就很难受了，想办法解决。async/await 是ES6语法，而小程序在开发的时候开发工具一般都会默认开启”ES6转ES5”这样的功能，而ES5是不认识 async/await 函数的，所以就报错了，看到这里的朋友可以直接试试，在微信开发工具中右上角的详情按钮点开会弹出功能窗，在里面将”ES6转ES5”的勾线框去掉，就会发现 async/await 就好使了。
  当然这并不是最好的办法，因为我们还是需要微信帮我们将ES6语法转成ES5语法的，这也是为了兼容大多版本比较低的机器，所以上述的第一个办法并不可取。上网查询了资料，说是少了regenerator-runtime的库，那我们就去加上这个库。

  • 随便新建一个文件夹，命令行进到该文件夹目录，执行 npm init
  • 看到生成了 package.json 文件再执行 npm install
  • 然后就会在目录中生成node_modules文件夹，打开这个文件夹，在里面找到名为regenerator-runtime的文件夹，将其拷贝到自己的微信小程序目录下
  • 在需要使用 async/await 的页面添加 import regeneratorRuntime from '../../libs/regenerator-runtime/runtime.js'; （”../../libs”是我的目录，请按照自己的目录引入）

      完了之后再运行，发现完美通过，不报错了，并且也实现同步的效果。
